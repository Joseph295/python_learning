# 写作规范（STYLE_SPEC）— 所有章节必须遵守

> 这份规范是整本手册的"黄金标准"。每一章（无论谁写）都必须严格按此结构、深度、风格产出，保证全书一致。

## 读者画像
- **Java 功底扎实**（懂 JVM、GC、字节码、并发、Spring 生态），**Python 近乎零基础**。
- 目标方向：**后端 API 服务** + **数据 / AI / 脚本自动化**。
- 学习策略：**快讲他已会的概念（OOP/控制流），慢讲"形似神不同"的差异**，篇幅花在差异和底层。

## 每章固定六段式结构（顺序不可变）

```
# NN · 标题

> 一句话导语：本章解决什么、为什么对 Java 老手重要。

## ① Java 对照表
   一张表，把本章概念逐条映射到 Java 对应物 + 一句差异说明。让读者 5 秒建立映射。

## ② 设计理念（为什么 Python 这么设计）
   讲 WHY。Python 的取舍、哲学动机、与 Java 设计目标的对比。不写 API。

## ③ 实现细节（底层怎么实现的）★本书重点★
   讲 HOW（底层）。CPython 层面的数据结构、算法、复杂度、内存布局、字节码、源码级机制。
   能画结构图/写复杂度表就写。对照 JVM 的对应实现。这是与普通教程拉开差距的部分，必须深。

## ④ 使用方法与最佳实践
   讲 HOW（应用层）。地道（Pythonic）写法、常用 API、工业界惯例、何时用何时不用。
   代码示例必须可运行、带关键输出注释。

## ⑤ ⚠️ 易错点（Java 思维翻车）
   三列表格：| Java 习惯 | 翻车现场 | 正确做法/认知 |。每条都是真实陷阱。

## ⑥ 练习题
   三档：**基础理解**（2题）→ **找坑改错**（1-2题，给一段 Java 味/有坑的代码让读者揪问题）→ **综合实战**（1题，贴近后端/数据场景）。
   末尾 `### 参考答案` 给出每题解析。
```

## 风格与硬性要求
- **语言**：简体中文讲解，代码与标识符英文。
- **Python 版本**：针对 3.12+，用到 3.10/3.11/3.12+ 新语法时标注最低版本。
- **代码**：全部可直接运行；关键行加 `# 输出: ...` 或 `# → ...` 注释；示例贴近后端/数据场景，不要 `foo/bar` 空壳。
- **深度基线**：以 `04_运行时与内存模型.md` 和 `05_CPython内部.md` 为深度样板。每章 ③ 实现细节段必须有真东西（数据结构/复杂度/字节码/源码机制），不能泛泛而谈。
- **Java 对照贯穿全章**，不只在 ① 段。处处用"在 Java 里你……，在 Python 里……"。
- **交叉引用**：相关主题用 `[NN 章名](NN_文件名.md)` 链接。
- **篇幅**：宁深勿浅，单章充实（参考 04/05 的体量）。不灌水、不重复其他章已讲的。
- **不 Pythonic 反面教材**：易错点和找坑题里要放真实的"Java 味 Python 代码"。
